# 数税云到聚兴 ERP 映射

## 映射原则

1. 数税云是税务事实源。
2. ERP 是业务事实源：合同、出库、入库、缺口、审批。
3. 金蝶是记账层：判断发票是否已入账、凭证关系是否闭环。
4. 飞书是唯一审批层：旧 ERP 开票申请不回主线。

## 一级菜单映射

| 数税云一级菜单 | ERP 当前位置 | 差距 |
|---|---|---|
| 销项管理 | 开票核对 → 我要核对/查账 → 销项管理 | 手工开票票面已可用并冻结；开票记录/订单/红字等需继续补接口和状态 |
| 进项管理 | 开票核对 → 我要核对/查账 → 进项管理 | 手工勾选、统计确认、认证结果已进 P0；真实勾选/确认动作需 L3 门禁 |
| 票据中心 | 开票核对 → 我要核对/查账 → 票据中心 | 发票池为 P0 主入口；签收、查验、任务、下载中心继续进 P1 |
| 分析看板 | 开票核对 → 我要核对/查账 → 分析看板 | 6 个二级页必须独立保留，已形成 P0 page spec |
| 基础信息 | 分散在客户/供应商/税编/额度配置 | 需要补完整资料库视觉和数据来源 |
| 系统设置 | ERP 暂无完整映射 | 只采集与数电、企业、权限相关部分 |

## P0 字段血缘基线

| 字段族 | 数税云来源 | ERP 来源 | 金蝶来源 | 保存/展示策略 |
|---|---|---|---|---|
| 发票主信息 | 发票池、开票记录、认证结果 | 本地税票镜像、业务回挂表 | 凭证号、入账日期、科目 | 数税云为事实源，ERP 保留镜像和回挂关系 |
| 购销方信息 | 发票抬头、税号、地址电话、银行账号 | 客户/供应商资料、合同客商 | 金蝶客商 | 页面展示以数税云票据为准，ERP 客商用于匹配 |
| 商品明细 | 发票明细、税收分类编码 | 合同商品、出库/入库明细、ERP 商品资料 | 存货/科目 | 税票商品和业务商品保留映射，不强行覆盖 |
| 金额税额 | 发票金额、税额、价税合计 | 合同金额、出入库金额、缺口金额 | 凭证金额、税额 | 通过差异队列展示，不自动抹平 |
| 认证/用途 | 用途状态、认证结果、统计确认 | ERP 批次、留抵测算 | 进项入账状态 | L0 查询可接；L3 真实动作必须门禁 |
| 额度 | `/prod-api/bussiness/credit/creditInfo/1` | `/api/tax-invoices/issue-credit?type=1` 代理 | 无 | 未配置数税云网页登录态时显示 `--`，不得写死假额度 |
| 审批 | 无 | 飞书审批、preview hash、幂等键 | 无 | 真实开票只允许飞书审批通过后执行 |

## 不允许恢复的旧链路

```text
合同 → invoice_application → 本地申请审批 → 数税云草稿
```

新链路固定为：

```text
合同/出库缺口
→ tax_invoice_issue_draft
→ 数税云票面
→ 保存/预览
→ 飞书审批
→ 审批通过真实开票
```
